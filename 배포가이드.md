## 남은 단계 (직접 하셔야 하는 것)

### 1. Apple Developer Program 가입 — $99/년
1. https://developer.apple.com/programs/enroll/ 접속
2. Apple ID로 로그인 후 개인(Individual) 자격으로 가입
3. 결제 후 승인까지 보통 1~2일

### 2. GitHub에 프로젝트 올리기
클라우드 Mac 빌드를 위해 필요합니다.
1. https://github.com/new 에서 새 리포지토리 생성 (Private 추천)
2. 이 폴더에서:
   ```
   git init
   git add .
   git commit -m "도트매트릭스 iOS 앱"
   git remote add origin https://github.com/<아이디>/<리포지토리>.git
   git push -u origin main
   ```

### 3. Codemagic 연결 (Mac 없이 빌드)
1. https://codemagic.io 에서 GitHub 계정으로 가입 (무료 티어: 월 500분)
2. 리포지토리 연결 → `codemagic.yaml` 자동 인식
3. **Teams → Integrations → App Store Connect**에서 API 키 연결
   - App Store Connect(https://appstoreconnect.apple.com) → 사용자 및 액세스 → 통합(Integrations) → API 키 생성
   - 키 이름을 `codemagic`으로 지정 (codemagic.yaml과 일치해야 함)
4. 빌드 실행 → 성공하면 TestFlight에 자동 업로드

### 4. App Store Connect에서 앱 등록
1. https://appstoreconnect.apple.com → 나의 앱 → + 새로운 앱
   - 번들 ID: `com.yunhee.dotmatrix` (developer.apple.com → Identifiers에서 먼저 등록)
   - 이름: 도트매트릭스 (또는 원하는 이름, 전 세계에서 고유해야 함)
2. 앱 정보 입력:
   - 스크린샷: TestFlight로 아이폰에 설치한 뒤 캡처 (6.9" / 6.5" 크기 필수)
   - 설명, 키워드, 카테고리 (유틸리티 또는 그래픽/디자인)
   - 개인정보처리방침 URL: `privacy.html`을 GitHub Pages 등에 올린 주소
   - 개인정보 수집 항목: "수집 안 함" 선택
3. TestFlight 빌드를 선택 → 심사 제출

### 5. 심사 (1~3일)
- 거부되면 사유가 메일로 옴 — 흔한 사유는 "4.2 최소 기능성"
- 거부 시 대응: 앱만의 기능(햅틱 피드백, 더 많은 도구 등)을 추가해 재제출

## 참고 사항

- **번들 ID 변경**: `capacitor.config.json`과 `codemagic.yaml` 두 곳의 `com.yunhee.dotmatrix`를 바꾸고 `npx cap sync ios` 실행
- **앱 수정 후 재배포**: `www/index.html` 수정 → `npx cap sync ios` → git push → Codemagic 빌드
- **아이콘 재생성**: `powershell -File make-icon.ps1` (assets/ 폴더에 생성됨)
- 아이콘을 iOS 프로젝트에 적용하려면 Codemagic 빌드 전에 `@capacitor/assets` 실행이 필요:
  ```
  npx @capacitor/assets generate --ios
  ```
